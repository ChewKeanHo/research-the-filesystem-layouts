# `/etc`

[![banner](/.internals/trademarks/banner_1200x100.svg)](#)

This is the base directory for housing critical and operating system (OS)'s
distributor supplied configuration files to function properly and minimally
without any mounting (e.g. `/usr` is not mounted or absent). This means it can
operate in `Single-User` mode in BSD realm or `Emergency Mode` in Linux realm.

The goal is to have minimally sufficient programs enough for basic
functionalities to perform critical tasks like mounting `/usr` UNIX System
Resources directory for OS capabilities extension, performing self-rescue, or
straight up being operational in this resources constraint environment such as
but not limited to OpenWRT embedded router.

Due to its processing nature, one **MUST** carefully work here to prevent any
data poisoning or losses.

All files here are available to all users to read but **ONLY** available to
specific user with permission, all sysadmins (user in `wheel` group), and `root`
account to create, update, and delete.

Generally, you **SHOULD ONLY** place programs and applications that are very
critical in the early booting stage here without conflicting with existing POSIX
compliant programs.

In all Linux-based OSes, this is the main directory for the OS configurations.
Many of them does not practice the use of `/usr/local/etc` layer.

In FreeBSD, it practices the use of `/usr/local/etc` so use that instead.

In Apple `MacOS`, this directory is facilitated mainly for supporting BSD
inter-compatibilities purposes only. `MacOS` does not not really use and depend
on it. Also this directory is part of the `local domain`.

For FreeBSD, it has the following notable configuration files that can make or
break the OS. Some already deployed the `.d` directory configurations strategy
so their existences are optional. Known configurations are:

```
/etc/
  auto_master         - auto-mount configurations (refer `automount` manual).
  crontab             - root's cron table.
  devfs.conf          - boot time device configuration.
  freebsd-update.conf - base OS updater configuration.
  fstab               - static filesystem configuration (refer `fstab` manual)
  hosts               - database of local hosts if no network nameserver is
                        running.
  inetd.conf          - configuration for BSD heritage internet server (refer
                        `inetd manual).
  localtime           - local timezone configuration (refer `ctime(3)` manual).
  login.conf          - login class capability database (refer `login.conf(5)`
                        manual.
  machine-id          - local system machine UUID for `dbus`.
  motd.template       - message display upon tty login (refer `motd(5)` manual).
  pf.conf             - Packet filter firewall (refer `pf(4)` manual).
  rc.conf             - system and daemon configuration (refer `rc.conf`
                        manual).
  resolv.conf         - DNS configuration (refer `resolv.conf`)
  resolvconf.conf     - DNS configuration  manager  configuration, often
                        generated by local-unbound (refer `local-unbound(8)` or
                        `resolvconf(8)` manuals).
  sysctl.conf         - kernel state defaults (refer `sysctl.conf` manual).
  syslog.conf         - system log information.
  ttys                - tty creation configuration (refer `getty` manual).
  wpa_supplicant.conf - client wifi configuration (refer
                        `wpa_supplicant.conf(5)` manual).
```




## Naming Conventions

[![banner](/.internals/trademarks/banner_1200x100.svg)](#)

It is a practice to house the files using `trademark` and `product`
sub-directories pattern. This can significantly reduces the naming collision for
common names.

The use of `.d` configuration directory strategy also reduces the configuration
complexities and making maintainability easier by just updating 1 configuration
per file. The program would loop through the directory and parse everything.

Here are the examples:

```
/etc/
  trademark/
    product/
      settings.d/
        setting1.conf
        setting2.conf
        ...

OR

/etc/
  product/
    settings.d/
      setting1.conf
      setting2.conf
      ...
```
